GET /patients/search

Makes search from all patients with given criteria. Either none or all search parameters can be set.

Request Parameters
name type description default constraints multivalued
firstNamePrefix query first name starts with condition, or null if not used as criterion     no
fullNameContains query patient's full name contains string condition, or null if not used as criterion Full name search overrides more specific lastNamePrefix, firstNamePrefix, userNamePrefix searches. A little bit unexpectedly full name contain search also include also match in case search string is part of patient's SSN i.e. henkilötunnus.     no
lastNamePrefix query last name starts with condition, or null if not used as criterion     no
list query optional patient list id(s), if not present patients are retrieved from global register   int yes
pageIndex query page index starting from which client wants to list patients 1 int no
pageSize query defines page size i.e. maximum amount of patients returned in on query, if page size is set, then also pageIndex parameter must be defined. If not set, returning all the patients matching given conditions. 50 int no
ssnPrefix query social security starts with condition, or null if not used as criterion     no
unit query optional unit ids, if present patient lists for given units are retrieved, otherwise login unit is assumed   int yes
Response Body
media type data type description
application/json PatientList (JSON) patients found with given criteria
