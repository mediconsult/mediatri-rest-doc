CoreAuthenticationDTO Data Type

DTO object for authentication purposes. This DTO is returned when logging in with REST.

Properties
name data type description
username string Username in core. This in not unique ID. Usually maps to Omistaja.Tunnus but it's not guaranteed (with Medinet-customer login this can be anything!).
roles object Roles grouped by unit id. Key is the unitId and value is @see UserRoleInfo Maps to UserRoleInfo
units array of Unit Unit map with units long name as key
activeRole number Returns roleId for current session. If roleId is null you must change role+unit using authentication-resource.
activeUnit number Returns unitId for current session. If unitId is null you must change role+unit using authentication-resource.
firstNames string Authenticated users first names. Can contain multiple names separated by spaces.
lastName string Last name of the authenticated user.
success boolean boolean true if authentication was successful. If false, errorMessage is provided in message-key.
message string Error message for authentication if authentication has failed
patient boolean true if authenticated user is patient-type.
userId number
developer boolean
componentRights array of ComponentRightDTO
viewConfiguration ViewConfiguration
