GET /hoitotyo/{id}/startShift

Starts shift by creating a new hoitotyo shift instance.

Request Parameters
name type description constraints
id path patient id int
copyPermanentRecors query boolean
referenceTime query reference time we want to start shift for. Parameter is optional and if omitted then current time is used. Reference time can be set to past, but not to the future. Time format without offset '2010-06-30T01:20' and with offset '2010-06-30T01:20+02:00'.  
Response Codes
code condition
400 Bad Request: if values were invalid e.g. in case trying to start shift in future time, or time format was invalid.
409 Conflict, shift has been already started.
401 Unauthenticated: The request requires user authentication.
403 Forbidden: User is authenticated but doesn't have enough priviledges to access this data. Re-login will not help and the request SHOULD NOT be repeated.
500 Unexpected error
Response Body
media type data type description
application/json Hoitotyo (JSON) hoitotyo dto for the created hoitotyo shift instance

#startShiftDataEdit(int, int, String)

POST /hoitotyo/{id}/{shiftId}/update

Updates hoitotyö shift data into the database. Client must mark all edited hoitotyömerkintas with Hoitotyomerkinta.isModified() : boolean flag, therefore only changed data rows are updated into database by creating new version on them. Note 1: #startShiftDataEdit(int, int, String) must be called prior calling this method. Note 2: This method only updates shift data i.e. hoitotyön merkintäs, not metadata for shift itself.

See Also
#startShiftDataEdit(int, int, String)
Request Parameters
name type description constraints
id path PatientID int
shiftId path Hoitotyo ID int
lockAcquireTime query optional parameter, if given identifies edit lock to be automatically released after shift data is updated to database.  
Request Body
