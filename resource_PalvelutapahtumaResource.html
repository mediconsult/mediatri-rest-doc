POST /patients/{patientId}/careRelation

Selects this palvelutapahtuma for proper care-relation to patient. This is logged and added to session, each request requiring palvelutapahtuma should use this Id. Response is text/plain since there is nothing more to respond that http-status 200. This also creates new hoitosuhde for patient using given palvelutapahtuma, or updates exising hoitosuhde data, if hoitosuhde was already existing.

Request Parameters
name type description constraints
palvelutapahtumaId form palvelutapahtuma id int
specialReasonCode form "Special reason" must be specified when care relation is made with users self-created encounter event. Code comes from 'THL- Asiakastietojen katselun erityinen syy" versiosta "1.2.246.537.6.240.2012' code system, but only a subset of those codes are used (listed in PalvelutapahtumaService#specialReasonCodes). The related optionalReasonText reason text parameter, coming from a client, does not one-to-one match with national code system descriptions. Therefore internal arrays PalvelutapahtumaService#specialReasonCodes and @see PalvelutapahtumaService#specialReasonTexts are used to validate texts.  
specialReasonText form One of the texts from pre-defined set of @see PalvelutapahtumaService#specialReasonTexts.  
patientId path patient id int
Response Codes
code condition
400 Bad Request: if values were invalid
401 Unauthenticated: The request requires user authentication.
403 Forbidden: User is authenticated but doesn't have enough priviledges to access this data. Re-login will not help and the request SHOULD NOT be repeated.
410 Gone: Patient has been removed OR palvelutapahtuma has been removed
412 palvelutapahtuma is not suitable for care relation.
500 Unexpected error
Response Body
media type data type description
application/json Hoitosuhde (JSON) Hoitosuhde dto with information about current hoitosuhde
