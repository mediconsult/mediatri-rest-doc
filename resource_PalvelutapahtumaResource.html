GET /palvelutapahtuma/aktiivinen/{id}

This method returns hoitosuhde for given patient for current logged in user.

POST /palvelutapahtuma/select

Selects this palvelutapahtuma for proper care-relation to patient. This is logged and added to session, each request requiring palvelutapahtuma should use this Id. Response is text/plain since there is nothing more to respond that http-status 200.

Request Parameters
name type description constraints
palvelutapahtumaId form palvelutapahtumaId int
patientId form patient id int
Response Codes
code condition
400 Bad Request: if values were invalid
401 Unauthenticated: The request requires user authentication.
403 Forbidden: User is authenticated but doesn't have enough priviledges to access this data. Re-login will not help and the request SHOULD NOT be repeated.
410 Gone: Patient has been removed OR palvelutapahtuma has been removed
412 palvelutapahtuma is not suitable for care relation.
500 Unexpected error
Response Body
media type data type description
application/json Hoitosuhde (JSON) Palvelutapahtuma instance
